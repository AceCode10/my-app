IP & MAC Addresses

IP Address — Internet Protocol Address

A unique number assigned to every device on a network.

  • IPv4: 32-bit address — e.g. 192.168.1.1
  • IPv6: 128-bit address — allows many more unique addresses — e.g. 2001:0db8:85a3::8a2e:0370:7334
  • IPv6 introduced because IPv4 addresses are running out

MAC Address — Media Access Control

A unique identifier assigned to a Network Interface Card (NIC).

  • It is a hardware address — does not change (unlike IP)
  • First 6 hex digits = manufacturer's code
  • Next 6 hex digits = device serial number
  • Used by switches to send data to the correct device

Routers & Data Packets

Router

A device that enables data to be sent between different types of networks. Can connect via cables or wirelessly.

Functions:

  • Connects networks and devices to the internet
  • Stores computer addresses in a routing table
  • Routes data packets to correct destination
  • Checks if packet is for its own network or forwards it

Data Packets

A small chunk of data sent over a network. Contains:

  • Actual data — the payload
  • Destination address — where it's going
  • Source address — where it came from
  • Error checking bits — to detect corruption

Large files are split into many packets and reassembled at the destination.

Common Network Devices

DeviceFunctionKey Feature
NIC
Network Interface Card		 Allows a device to connect to a network. Converts binary data to electrical signal. Usually integrated into motherboard. Each has a unique MAC address.
Hub Connects devices to form a LAN. Receives a data packet and sends to all connected devices. Not secure — broadcasts to everyone. Creates unnecessary network traffic.
Switch Intelligent hub. Connects devices in a LAN. Sends data to the relevant device only using MAC lookup table. More secure than hubs. Uses MAC addresses to identify correct destination.
Bridge Connects one LAN to another LAN using the same protocol. Cannot communicate with external networks like the internet. Scans MAC addresses (unlike routers which use IP). Only 2 ports.
Hub vs Switch: Both connect LAN devices using data packets. Hubs send to ALL devices; Switches send to the RELEVANT device only. Switches are more secure and efficient.

Wi-Fi & Bluetooth

Both use wireless communication — electromagnetic radiation as carrier of data transmission.

Wi-Fi

Used for: full-scale networks; when data transfer rate is crucial; higher range; better security

  • Frequency: 2.4 – 5.0 GHz
  • Much faster data transfer rate
  • Higher geographical range
  • Encryption: WPA and WEP
  • Number of devices depends on router

Bluetooth

Used for: transferring data between devices very close together; low-bandwidth applications; when speed isn't crucial

  • Frequency: 2.4 GHz
  • Slower data transfer rate
  • Shorter range
  • Encryption: key-matching
  • Maximum 7 devices simultaneously

Cloud Computing

Cloud Computing — data stored on remote servers accessed online. Three types:

Public Cloud

Customer and cloud provider are different companies. Shared infrastructure.

Private Cloud

Customer and provider are the same organisation. Data stays internal.

Hybrid Cloud

Combines both. Sensitive data stays private; less sensitive uses public cloud.

Advantages:

  • Scalable — services scaled up/down as needed
  • No need to purchase expensive hardware
  • Globally accessible — enables remote working
  • Almost unlimited storage
  • Full data recovery from backup
  • Files synced across multiple devices
  • No need to carry storage devices

Disadvantages:

  • Requires stable internet connection always
  • Vulnerable to security breaches
  • Dependent on provider for reliability
  • Can become expensive as storage/bandwidth increases

Network Types: LAN, WLAN & WAN

LAN Local Area Network

Contained within a single building or small area. Uses hubs/switches; one hub/switch connects to router.

Adv: Centralised management of updates/backups; resource sharing (printers); firewall/antivirus security

Dis: Hardware failure stops network; more prone to attacks; slow if traffic heavy; requires maintenance

WLAN Wireless LAN

Devices connect wirelessly via Wireless Access Points (WAPs). Popular in public areas (malls, cafes).

Adv: Connect anywhere in WAP range; flexible (indoor/outdoor); additional WAPs added easily; wireless printing

Dis: Walls reduce signal strength; high traffic reduces speed; interference from other devices; security threats via signal interception

WAN Wide Area Network

Formed by several LANs connected via routers. Expensive but more enhanced security. Needs routers, modems, proxy servers.

Adv: Connects geographically dispersed locations (different cities/countries); scalable; centralised monitoring

Dis: Expensive to set up and maintain; latency over long distances; relies on external service providers

4.2 Network Security Threats

Connection to a network exposes users to several threats:

ThreatDescriptionPrevention
HackingUnauthorised access to a computer system. Can lead to identity theft or misuse of personal information.Strong passwords, firewalls, two-factor authentication
PhishingGetting sensitive information (usernames, passwords, card details) by mimicking a trusted webpage — user must take action (click link).Anti-phishing filters; never click suspicious links
PharmingMalicious code installed on user's computer or web server — redirects user to fake website without any action from the user.Keep software updated; use reputable DNS servers
VirusesProgram code that replicates itself — deletes or corrupts files and causes computer to malfunction.Up-to-date antivirus software; avoid unknown downloads

Passwords & Authentication Methods

Passwords

Common security measure — often used with username or email address.

Weak: iloveict

Strong: 1lov3ICT#

To enhance security:

  • Change regularly in case obtained illegally
  • Use uppercase, lowercase, numbers & special characters
  • Run anti-spyware regularly to prevent interception

Other Authentication Methods

  • Zero Login — eliminates username/password; uses biometrics for seamless access
  • Biometric Recognition — fingerprints, iris scans, voice recognition
  • Magnetic Stripes — encoded data on cards read by magnetic stripe readers
  • Smart Cards — portable devices with integrated circuits for banking, telecoms, access control
  • Physical Tokens — solid objects generating one-time passwords (OTPs) with embedded clocks
  • Electronic Tokens — apps on smartphones generating time-limited OTPs

Anti-Malware Software

Designed to detect, prevent and remove malicious software. Installed on the system and runs in the background.

Standard Features

  • Compares scanned files against a database of known threats
  • Real-time scanning of files and downloads
  • Regular updates to maintain current threat database
  • Quarantine of infected files

Quarantine & Heuristic Checking

  • Quarantine: isolates threats for automatic deletion or user review — avoids false positives being deleted unnecessarily
  • Heuristic Checking: instead of using only a known virus database, looks for behavioural patterns in files to identify new/unknown threats

Electronic Conferencing — Video Conferencing

Video Conferencing — audio and video communication in real-time as a substitute for face-to-face meetings.

Hardware & Software Required

  • Webcams, large monitors/TV screens
  • Microphones, speakers
  • Webcam & microphone software drivers
  • CODEC (coder and decoder)
  • Echo cancellation software

Advantages:

  • Easy to access documents or bring in 'experts'
  • Can be held at short notice
  • Reduced travelling costs
  • No hotel/venue costs
  • Safer than travelling to unsafe locations

Disadvantages:

  • Potential time lag/delays when talking
  • Images can jerk
  • Expensive initial setup
  • Time zone differences cause problems
  • Completely reliant on stable internet connection

Audio & Web Conferencing

Audio Conferencing

Meetings using audio (sound) technology — also called phone conferencing. Uses normal telephone network.

Hardware: computer with built-in mic/speakers; external mic/speakers; internet phone; standard phone

Adv: Join from any location; no travel expenses; saves time

Dis: No visual — hinders communication; limits engagement; technical issues (poor audio, background noise)

Web Conferencing

Uses internet for conference calls (webinars/webcasts). Multiple computers interconnected. Operates in real-time.

Meeting types: business meetings; presentations; online education/training

Adv: Video adds immersion; screen sharing for presentations; interactive features (chat, polling, Q&A)

Dis: Relies on internet; susceptible to distractions; learning curve for new users

Topic 4 — Key Points

4.1 Networks

  • IP = unique logical address (changes) · MAC = unique hardware address (permanent)
  • Router routes packets between networks using routing tables
  • Hub → all devices · Switch → correct device only
  • Wi-Fi: faster, longer range · Bluetooth: close range, 7 devices max
  • Cloud: public / private / hybrid — scalable but needs internet
  • Internet (global open) · Intranet (private internal) · Extranet (selected external access)
  • LAN (building) · WLAN (wireless) · WAN (multiple LANs, long distance)

4.2 Security & Communication

  • Threats: Hacking · Phishing (needs action) · Pharming (no action needed) · Viruses
  • Passwords: use uppercase, lowercase, numbers, symbols; change regularly
  • Authentication: Zero login · Biometric · Smart cards · Physical/Electronic tokens
  • Anti-malware: detect, prevent, remove; real-time scanning; heuristic checking
  • Video conf: real-time audio+video; needs CODEC
  • Audio conf: phone-based; no visual element
  • Web conf: internet-based; screen sharing; interactive
